Отчёт по бэкенду приложения тайм-менеджмента

FastAPI · PostgreSQL · Alembic · JWT · слоистая архитектура

1. Цель работы

Реализовать серверную часть веб-сервиса для планирования времени: проекты и задачи с приоритетами и статусами, учёт затраченного времени, напоминания, дневные планы, теги и связь записей времени с тегами по схеме многие-ко-многим с дополнительным атрибутом на связи (relevance_score).

2. Используемые технологии

  • Python 3.13+, асинхронный стек
  • FastAPI — HTTP API, валидация через Pydantic
  • SQLAlchemy 2 — ORM, async-движок (asyncpg)
  • PostgreSQL — хранение данных
  • Alembic — версионирование схемы БД
  • JWT (python-jose) — аутентификация после логина по OAuth2 password flow

3. Архитектура проекта

Код бэкенда расположен в каталоге students/k3341/Skoblilova_Viktoria/Lr1/gateway/ и разделён на слои:

СлойКаталогНазначение
Моделиmodels/Таблицы и связи SQLAlchemy
Схемыtemplates/base_models/Модели запросов/ответов Pydantic
Логикаcore/auth, core/time, core/usersОперации с БД, проверки владельца, функции *_implementation

Такое разделение упрощает сопровождение: изменение правил бизнес-логики не смешивается с описанием маршрутов и структуры таблиц.

Логические сущности отображаются на таблицы; часть имён таблиц исторически совпадает с прежним «финансовым» шаблоном — это сознательно сохранено для совместимости с цепочкой миграций.

5. API (обзор)

Авторизация — /auth

  • POST /auth/login — выдача JWT (форма OAuth2: email в поле username)
  • POST /auth/register — регистрация
  • POST /auth/change-password — смена пароля (Bearer)
  • GET /auth/me, GET /auth/users — профиль и списки в зоне auth

Пользователи

  • GET /users, GET /users/{id}
  • GET /users/{id}/details — агрегированный ответ с вложенными сущностями тайм-менеджера

Домен времени (все с Bearer)

Ресурсы: /projects, /tasks, /time-entries, /reminders, /daily-plans, /tags. Поддерживаются операции CRUD по REST-подобным соглашениям; отдельно POST /time-entry-tags для привязки тега к записи времени.

6. Запуск и развёртывание

Рекомендуемый способ — Docker Compose из каталога students/k3341/Skoblilova_Viktoria/Lr1: сервисы gateway и gateway-db, переменные из gateway/.env (шаблон — gateway/example.env).

При старте приложения выполняется применение миграций и инициализация демо-данных (модуль schedulers.data_init). Для отладки возможен полный сброс схемы через флаг FORCE_RESET_DATABASE в настройках (только для среды разработки).

7. Выводы

Собран работоспособный бэкенд с разделением на слои, JWT-аутентификацией и доменной моделью тайм-менеджмента, включая ассоциативную сущность M2M с дополнительным числовым полем на связи. Документация OpenAPI доступна на работающем сервисе по адресу /docs (при включении в точке входа приложения).
